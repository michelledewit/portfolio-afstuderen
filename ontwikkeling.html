Ontwikkeling
Hoe kan een een werkende prototype worden uitgewerkt met de frameworks die binnen Brik worden gebruikt waardoor de kwalitatieve leads inzichtelijk zijn in Craft CMS?
Inleiding
In de ontwikkelingsfase stond de deelvraag “Hoe kan een een technisch prototype worden uitgewerkt met de frameworks die binnen Brik worden gebruikt waardoor de kwalitatieve leads inzichtelijk zijn?” centraal. Om deze vraag te beantwoorden is er gestart met het uitwerken van de Front-end ontwikkeling. Deze Front-end is gekoppeld aan Craft CMS waardoor de content van de componenten te vullen zijn vanuit het back-end systeem. Nadat de basis van de landingspagina stond heb is een eerste MVP uitgewerk waarin de lead score van een MSCH-profiel inzichtelijk is en op basis daarvan content wordt getoond of verborgen. Uitbreiding mvp In de volgende hoofdstukken is het proces van de technsiche realisatie uitgewerkt.
Front-end landingspagina uitwerken
Na het uitwerken van het ontwerp is de de Front-end ontwikkelng gestart gebaseerd op het design van de landingspagina. In de Front-end ontwikkeling is gebruik gemaakt van het CSS framework TailwindCSS. Er is voor Tailwind gekozen aangezien dit binnen Brik in de bestaande website het gebruikte CSS framework is. TailwindCSS maakt gebruik van utility classes waardoor herbruikbare componenten kunnen worden gebouwd. Aan de HTML elementen is de CSS toegevoegd door middel van het meegeven van Tailwind classnames. De HTMl en CSS is op deze manier overzichtelijk in één bestand samengevoegd.
Elk component van het ontwerp zoals een CTA of een testimonials, is op deze manier voor de MSCH-profielen uitgewerkt. Hierdoor krijgt elk profiel zijn eigen variatie die aansluit bij de informatiebehoefte waardoor er voor bijvoorbeeld de CTA vier variaties zijn gebouwd. Zie in de onderstaande afbeelding het CTA element voor de humanistische bezoeker.
Front-end koppelen met Craft CMS
Creëren pagina's
Binnen Brik wordt voor digitale webapplicaties gebruik gemaakt van het content management systeem Craft CMS. Craft CMS is volledig naar eigen hand in te stellen waarmee de content voor de landingspagina's kan worden gevuld in handgemaakte templates met Twig en PHP. Na het verbinden van de Front-end met Craft CMS is Craft ingericht. De eerste stap was het creëren van de landingspagina’s, conversiepagina's en bedankpagina's.
MSCH-profiel eigen versies per contentblok
De pagina's bestaan uit contentblokken. Elk component van het ontwerp (CTA, FAQ, etc.) heeft zijn eigen contentblok. Elk contentblok heeft vier versies, één versie per MSCH-profiel zodat de juiste versie van het contentblok kan worden geladen wanneer het profiel van een bezoeker is achterhaald. Een spontane bezoeker krijgt bijvoorbeeld een afbeelding te zien en een methodische weer niet, elk contentblok heeft zo zijn eigen velden die vanuit het CMS gevuld kunnen worden
Contentblokken
Een contentblok bestaat uit verschillende velden zoals het CTA element een titel, tekst, link en een afbeelding. Deze velden zijn middels Twig geïmplementeerd in de Front-end waardoor de content in het CMS per veld kan worden aangepast. Dit is voor elk contentblok op deze manier uitgewerkt.
Alpine JS
Voor het tonen van de juiste versie van een contentblok is gebruik gemaakt van Alpine JS. Alpine JS is het JavaScript framework wat gebruikt wordt door Brik in webapplicaties en wordt op aanraden van het Brik team toegepast op de landingspagina. Een belangrijke eigenschap van Alpine JS is het toepassen van dynamic loading. Met dynamic loading kunnen er elementen worden getoond of verborgen zonder dat de pagina opnieuw geladen hoeft te worden. Op deze manier kan de juiste contentvorm worden geladen op basis van de uitgevoerde acties van een bezoeker zodat de content aansluit bij het type doelgroepprofiel.
Lead scoring + Dynamische content MSCH-profiel
Eerste MVP Lead scoring & Dynamische content
Nadat de Front-end ontwikkeling is gekoppeld aan Craft CMS stond het uitwerken van een eerste MVP centraal waarin content wordt geladen die past bij het MSCH-profiel, gebaseerd op lead scoring. Op basis van een actie (klik op knop) krijgt het humanistische profiel een punt. Bij 3 punten wordt dan door middel van Alpine JS dynamische content geladen die aansluit bij het profiel.

Git
Voor versiebeheer van de technische ontwikkeling is de binnen Brik gebruikte management tool Gitlab toegepast. Hierdoor kan de code worden beheerd in groepsverband en kan er door middel van versiebeheer eenvoudig worden teruggevallen op oudere versies. Binnen de repository van het project heb is een branch aangemaakt speciaal gericht op het realiseren van de landingspagina. Hierin is de binnen Brik gebruikte naming convention (feature/naam-feature).
Tijdens de realisatie zijn de verschillende toevoegingen en aanpassingen binnen de code doorgezet (commits) naar de landingspagina branch. Elke commit bevat de binnen Brik standaard toegepaste naming convention: "type commit" + "uitgevoerde werkzaamheid".
